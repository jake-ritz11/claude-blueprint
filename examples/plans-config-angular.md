# Example: _plans-config.md for an Angular project

This shows how you might customize the Standards Files section of `_plans-config.md` for an Angular + Vitest project.

---

## Standards Files

Read the relevant `.ai/` standards files per the task type table below:

| File type             | Standards file                          |
| --------------------- | --------------------------------------- |
| `*.component.ts`      | `.ai/angular-components-standards.md`   |
| `*.service.ts`        | `.ai/angular-services-standards.md`     |
| `*.test.ts`           | `.ai/unit-testing-vitest.md`            |
| `*.css`               | `.ai/css-standards.md`                  |
| `*.directive.ts`      | `.ai/angular-coding-standards.md`       |
| Other `*.ts`          | `.ai/coding-standards.md`              |

Also check:

- `.ai/codebase-knowledge/README.md` for cross-cutting gotchas
- `.ai/<feature-name>.md` for feature-specific knowledge
