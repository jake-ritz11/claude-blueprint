# Example: _plans-config.md for a React/Next.js project

This shows how you might customize the Standards Files section of `_plans-config.md` for a React project.

---

## Standards Files

Read the relevant standards files per the task type:

| File type            | Standards file                         |
| -------------------- | -------------------------------------- |
| `*.tsx` (components) | `.cursor/rules/component-standards.md` |
| `*.ts` (hooks)       | `.cursor/rules/hooks-standards.md`     |
| `*.test.tsx`         | `.cursor/rules/testing-standards.md`   |
| `*.css` / `*.module.css` | `.cursor/rules/styling-guide.md`  |
| API routes           | `.cursor/rules/api-standards.md`       |

Also check:

- `docs/architecture/` for design decisions
- Component-level `README.md` files in feature directories
